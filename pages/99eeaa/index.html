单例

# 介绍

单例模式（Singleton Pattern）是 Java 中最简单的设计模式之一，属于创建型模式。该类负责创建自己的对象，同时确保只有单个对象被创建。不需要实例化该类的对象，可以通过类直接获得对象。

# 实现

单例的实现有很多种，这里介绍两种常用的：懒汉式 和 饿汉式 。

# 1. 饿汉式

  • 优点：在类装载的时候就完成实例化，避免了线程不安全问题。
  • 缺点：会造成内存的浪费。
class Singleton2 {
    private static Singleton2 singleton2 = new Singleton2();

    public static Singleton2 getInstance() {
        return singleton2;
    }
}

# 2. 懒汉式

  • 优点：起到了 Lazy Loading 的效果，不浪费内存。
  • 缺点：线程不安全。
class Singleton1 {
    private static Singleton1 singleton;

    public static Singleton1 getInstance() {
        if (singleton == null) {
            return new Singleton1();
        }
        return singleton;
    }
}
