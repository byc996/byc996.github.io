面向对象

# 封装 Encapsulation

将数据和和操作数据的方法绑定到同一个单元内（类）。通常会定义所有的成员变量为 private ，避免外部直接访问，通过 public 方法进行访问。这样就可以控制数据的访问权限。

# 继承 Inheritance

  • 子类拥有父类所有的属性和方法，但是 private 的属性和方法无法访问。
  • final 关键字修饰的类无法继承
  • super 关键字可以调用父类的 public 方法和属性（包括构造函数），this 关键字可以调用当前对象的属性和方法
  • Java 中类不支持多继承，接口可以多继承

笔记

Java 中创建子类对象不会创建父类对象！参考 (opens new window)

  • 创建对象指的是在堆区开辟空间
  • 编译器在运行子类构造器之前，必须先执行父类构造器；且调用父类构造器的语句必须在子类构造器的第一行。
  • 构造方法的作用是为堆区中的对象的属性初始化，不是创建对象。

# 多态 Polymorphism

什么是多态？同一个行为具有多个不同表现形式或形态的能力。

# 多态的分类

多态分为编译时多态（静态）和运行时多态（动态）。

# 多态的特点

父类引用变量指向子类对象

引用类型变量发出的方法调用的到底是哪个类中的方法，必须在程序运行期间才能确定；

多态不能调用 “只在子类存在但在父类不存在” 的方法；

如果子类重写了父类的方法，真正执行的是子类覆盖的方法，如果子类没有覆盖父类的方法，执行的是父类的方法。

# 抽象 Abstraction

Java 中主要通过抽象类和接口来实现抽象的。

# 抽象类

  • 抽象方法不能为 private ，必须使用 abstract 修饰
  • 不能有实现体，必须被子类重写

# 接口

  • 类能实现多个接口，接口可以继承多个接口
  • 接口不能为静态，但能有静态方法（必须有具体实现）
  • 实现类对象无法调用接口的静态方法，但子类对象能调用父类的非私有静态方法（但建议使用类。方法的形式调用）
  • 接口实现了松耦合，类之间互相不影响

# 接口 VS 抽象类

相同点：

  • 都不能被实例化
  • 都可以可以包括抽象方法（接口中加不加 abstract 关键字都一样）
  • 都可以有默认方法（接口在 Java 8 之后支持默认方法，使用 default 关键字）

不同点：

接口 抽象类
支持多继承 不支持多继承
所有成员都是公共的 public 成员可以是 private , protected , public
成员变量必须是 static and final 成员变量可以是静态或非静态
不能有构造函数 可以定义构造函数

# 函数式接口

函数式接口：只有一个抽象方法。（可以有多个默认方法）

@FunctionalInterface 注解不是必须的，但是如果加了注解而且有多个抽象方法，则会报错。

Java 8 中定义了 java.util.function 包，包含一些函数式接口，比如 Predicate , Consumer , Supplier , Function , 等等
