Marker Interface

# 什么是 Marker Interface?

标记接口（Marker Interface）是一种不包含任何方法声明的接口。

标记接口是计算机科学中的一种设计思路。编程语言本身不支持为类维护元数据。而标记接口则弥补了这个功能上的缺失 —— 一个类实现某个没有任何方法的标记接口，实际上标记接口从某种意义上说就成为了这个类的元数据之一。运行时，通过编程语言的反射机制，我们就可以在代码里拿到这种元数据。

Java 中常见的标记接口：

  • Cloneable: 可以对该类实例进行按字段复制。
  • Serializable：启动序列化功能，可以序列化或反序列化。
  • RandomAccess: 表明其支持快速（通常是固定时间）随机访问
  • …

    class Person {
    public void teach() {
    }
}

class Teacher extends Person implements MarkerInterface {
    @Override
    public void teach() {
        System.out.println("Teacher teach...");
    }
}

class Student extends Person {
    @Override
    public void teach() {
        System.out.println("Student teach...");
    }
}

  3. 通过反射调用方法，会判断实例是否实现了 MarkerInterface 接口，如果实现了才能调用 teach 方法，否则报错。

    public static void callTeach(Person person) {
    Class<?> teacherClass = person.getClass();
    Method declaredMethod = null;
    try {
        declaredMethod = teacherClass.getDeclaredMethod("teach");
        if (person instanceof MarkerInterface) {
            declaredMethod.invoke(person);
        } else {
            throw new RuntimeException("Object is not an instance of MarkerInterface");
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
}

public static void main(String[] args) {
    Teacher teacher = new Teacher();
    callTeach(teacher);
    Student student = new Student();
    callTeach(student);
}

笔记

instanceof 判断对象和类型之间的关系，是关键字，只能用于对象实例，判断左边的对象是不是右边的类 (包括父类) 或者接口 (包括父类) 的实例化。

# 总结

Java 中标记接口的主要用途是向 JVM 传达实现某些接口的类必须被授予某些特殊行为。标记接口从某种意义上说就成为了这个类的元数据之一，可以通过反射机制获取到这个元数据，从而判断对象时候实现了标记接口。

当一个类实现了 Serializable 接口时，这是一个标记接口，那么这就向 JVM 表明该类的对象可以被序列化。 同样，当一个类实现了 Cloneable Interface 时，它就向 JVM 表明该类的对象可以被克隆。
