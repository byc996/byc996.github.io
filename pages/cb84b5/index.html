统一的响应和异常处理

# HTTP 状态码

HTTP 状态码分类

分类 描述
100–199 信息响应
200–299 成功响应
300–399 重定向
400–499 客户端错误
500–599 服务器错误

HTTP 常见状态码（status code）参考 (opens new window)

状态码 名称 描述
200 OK 请求成功
201 Created 已创建
301 Moved Permanently 永久移动。请求的资源已被永久的移动到新 URI，返回信息会包括新的 URI，浏览器会自动定向到新 URI
400 Bad Request 客户端请求的语法错误，比如参数错误
401 Unauthorized 请求要求用户的身份认证，比如未登录
403 Forbidden 无权限，禁止访问
404 Not Found 找不到资源（页面）
500 Internal Server Error 服务器内部错误

# 自定义状态码

在 Java 中定义枚举类来定义 http 状态码，用来定义常用的状态码。

public enum Status {

    SUCCESS(200, "OK"),
    PARAMS_ERROR(400, "Request parameters error"),
    UNAUTHORIZED_ERROR(401, "Unauthorized: not logged in"),
    FORBIDDEN_ERROR(403, "Forbidden: no permission to access"),
    NOT_FOUND_ERROR(404, "Resource does not exist"),
    SERVER_ERROR(500, "Internal Server Error");

    private final int code;

    private final String message;

    Status(int code, String message) {
        this.code = code;
        this.message = message;
    }

    public int getCode(){
        return this.code;
    }

    public String getMessage() {
        return this.message;
    }
}

# 统一的返回响应

# 响应类

在前后端分离的架构下，后端的接口都是 RESTful 风格的。为了让前端使用方便以及统一风格，定义一个统一响应类作为所有响应的封装类是有必要的。

统一响应类主要包括三个字段：

  • code：响应码
  • message：响应信息包括错误信息
  • data：具体返回给前端的响应数据
import lombok.AllArgsConstructor;
import lombok.Data;
import lombok.NoArgsConstructor;

import java.io.Serializable;

@Data
@AllArgsConstructor
@NoArgsConstructor
public class BaseResponse<T> implements Serializable {

    private Integer code;

    private String message;

    private T data;


    BaseResponse(int code, String message){
        this(code, message, null);
    }

    BaseResponse(Status status) {
        this(status.getCode(), status.getMessage());
    }
    BaseResponse(Status status, T data) {
        this(status.getCode(), status.getMessage(), data);
    }
}

# 响应工具类

在 controller 中，当请求方法执行正常时调用 success 方法返回响应数据，而如果请求方法执行异常，则直接抛出异常交给全局异常处理器，全局异常处理器会捕获异常并调用 error 方法返回异常信息给前端。

public class ResponseUtils {

    public static <T> BaseResponse<T> success(T data) {
        return new BaseResponse<>(Status.SUCCESS, data);
    }

    public static BaseResponse error(int code, String message) {
        return new BaseResponse(code, message);
    }
    public static BaseResponse error(Status status) {
        return new BaseResponse(status);
    }

    public static BaseResponse error(Status status, String message) {
        return new BaseResponse(status, message);
    }

}

# 自定义异常类

public class BusinessException extends RuntimeException{

    private Integer code;

    public BusinessException(int code, String message){
        super(message);
        this.code = code;
    }

    public BusinessException(Status status) {
        super(status.getMessage());
        this.code = status.getCode();
    }

    public BusinessException(Status status, String message) {
        super(message);
        this.code = status.getCode();
    }

    public int getCode() {
        return this.code;
    }
}

# 全局异常处理器

@RestControllerAdvice + @ExceptionHandler 可以进行全局异常处理。

统一处理 HTTP 请求时（所有注解了 @RequestMapping 的控制器的方法）抛出的异常，根据不同的异常类型可以自定义处理的逻辑。

@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public BaseResponse<?> businessExceptionHandler(BusinessException e) {
        log.error("businessException: " + e.getMessage(), e);
        return ResponseUtils.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(RuntimeException.class)
    public BaseResponse<?> runtimeExceptionHandler(RuntimeException e) {
        log.error("runtimeException: ", e);
        return ResponseUtils.error(Status.SERVER_ERROR, e.getMessage());
    }
}

# Controller 参考示例

@RestController
@RequestMapping("/payment")
@Slf4j
public class PaymentController {

    @Resource
    private PaymentService paymentService;

    @GetMapping(value = "/get/{id}")
    public BaseResponse<Payment> getPaymentById(@PathVariable("id") Long id) {
        Payment payment = paymentService.getPaymentById(id);
        log.info("*********查询结果：" + payment);
        if (payment != null) {
            return ResponseUtils.success(payment);

        } else {
            throw new BusinessException(Status.NOT_FOUND_ERROR, "没有对应记录，查询id:" + id);
        }
    }
}
