# Java 集合框架结构

Java 集合主要分为两块 Collection 和 Map，Collection 接口主要有三个子接口：List, Set, Queue。参考 (opens new window)

# Collection

集合接口的方法：

# List

  • ArrayList：基于数组实现，可以随机访问 O (1)。插入，删除元素复杂度高 O (n)，非线程安全。
  • LinkedList：基于双向链表实现，可以快速插入和删除元素 O (1)，访问元素复杂度高 O (n)，非线程安全。
  • Vector：与 ArrayList 类似，但是线程安全，所有需要保证线程安全的方法都添加了 synchronized 关键字。

# Set

  • HashSet：基于哈希表实现，可以快速查找元素 O (1)，无序，非线程安全。
  • LinkedHashSet：基于哈希表和链表实现，可以快速查找元素 O (1)， 有序，非线程安全。
  • TreeSet：基于红黑树实现，查找元素复杂度 O (logN)，有序，非线程安全。

# Queue

  • LinkedList：基于双向链表实现，可以快速插入和删除元素 O (1)，访问元素复杂度高 O (n)，非线程安全。
  • PriorityQueue：基于堆实现，元素操作复杂度 O (logn) 或 O (n)，非线程安全。

# Map

Map 接口的方法：

Map 常用实现类：

  • HashMap：基于哈希表实现，无序，非线程安全。
  • LinkedHashMap：基于哈希表和链表实现，有序，非线程安全。
  • HashTable：与 HashMap 类似，无序，但是线程安全。
  • TreeMap：基于红黑树实现，有序，非线程安全。
