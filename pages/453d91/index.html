Github + PicGo + Typora 图床

为了统一管理 Github Repos README.md 中的图片，可以在 GitHub 中建一个仓库作为图床进行统一管理。本地使用 typora 编辑，使用 PicGoz 自动上传图片并生成外链，这样就不需要每个项目中都存放图片，更加简洁。另外在平时笔记中也可以使用 GitHub 存储笔记图片，这样在笔记迁移的过程中不要担心图片的丢失或者图片的路径不匹配问题。

# Github 创建 repo 并新建 token

# 1. 新建 public 仓库

仓库必须为 public，因为需要外部访问，否则只能写不能读。

# 2. 创建 token

打卡 settings -> developer settings -> personal access tokens -> token (classic) -> generate new token (classic)

只需要勾选 repo 即可。

# 配置 PicGo

PicGo 下载地址： https://github.com/Molunerfinn/PicGo/releases

图床设置选择 Github， 填入刚才生成的 Github token 即可。

设置完成后可以使用 PicGo 上传图片，这样图片就会上传到 GitHub 图床了。

提示

jsdelivr 是免费的且不限流量的 CDN， 可以存储些网站的静态资源，然后引用 CDN 地址，优化图片的访问速度。格式：https://cdn.jsdelivr.net/gh/<github-username>/<repo-name>

# 配置 Typora

那么 Typora 如何配置 PicGo 使用呢？

# 英文版（不推荐）：

打开 File->Preferences -> Image, 修改图片配置中的 image uploader 为 PicGo-core, 然后运行 Test Uploader 检查是否配置成功。

如果测试报错，可能是因为 Typora 找不到 PicGo 的配置文件导致的。所以我们需要打开两边的配置查看：

Typora: File->Preferences -> Image -> open config file -> config.json

PicGo: PicGo 配置 -> 打开配置文件 -> data.json

将 data.json 的内容拷贝到 config.json 中，这样就能找到配置了不会报错了。

注意

每次修改 PicGo 配置文件时，比如添加了新的插件，需要同步到 Typora 的配置文件。

PicGo-core有一个坑：picgo的插件不能生效，比如watermark水印。

# 中文版（推荐）：

image uploader 设置为 picgo (app), 然后设置 picgo 所在路径即可，而且插件生效，推荐！

笔记

PicGo.app 是 GUI 版的 PicGo-Core，仅提供中文版。这里使用 PicGo-Core。
https://support.typora.io/Upload-Image/#picgo-core-command-line-opensource

# 总结

至此，我们就可以在 Typora 直接粘贴图片并同时自动上传到 GitHub 图床中生成 jsdelivr 提供的外链，因为图片是以链接的形式保存在笔记中，以后迁移笔记的时候也不需要担心图片路径的问题了。
