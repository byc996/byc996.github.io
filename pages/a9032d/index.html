# 泛型

什么是泛型？

泛型的本质是参数化类型，也就是说所操作的数据类型被指定为一个参数。

泛型的意义

泛型的主要作用就是代码复用，同一套代码适用于不同的数据类型。

# 泛型上下限

  • <T extends Number> 或 <? extends Number> ：表示类型必须是 Number 的子类
  • <T super Cat> 或 <? super Cat> ：表示类型必须是 Cat 的父类

注意

<T extends Number> 中如果或者是接口也必须用 extends，比如 <T extends Serializable>

如果继承多个可以使用 "&" 符号，比如 <T extends Person & Serializable> ，这里需要注意接口必须放在后面，否则会报错。

# 泛型的分类

  • 泛型类
  • 泛型方法
  • 泛型数组

下面是不同类型各自的示例。

# 泛型类

// T: 任何类型的数据
class GenericClass <T> {
    T data;
    public GenericClass(T data) {
        this.data = data;
    }

    public void print() {
        System.out.println(data);
    }
}

public class GenericDemo {

    public static void main(String[] args) {
        GenericClass<Float> integerGenericClass = new GenericClass<>(23f);
//        GenericClass<String> integerGenericClass1 = new GenericClass<>("sdf");
        integerGenericClass.print();
    }
}

    public static <T, V> void print(T data1, V data2) {

# 泛型数组

如下如果想要打印不同类型的数据，我们不能使用 List<Object>，因为尽管 Integer 是 Object 的子类，但是 List<Integer > 并不是 List<Object > 的子类，所以这里必须使用 "?" 通配符 (wildcard) 代替 Object 来解决这个问题，描述任意类型的数组则可以用 List<?>。

public class GenericDemo {

    public static void printList(List<?> list) {
        System.out.println(list);
    }

    public static void main(String[] args) {

        List<Integer> integers = new ArrayList<>(Arrays.asList(1,2,3));
        printList(integers);

        List<String> strs = new ArrayList<>(Arrays.asList("hello", " wk ", "world"));
        printList(strs);
    }
}

笔记

? 同样可以使用上下界，比如 <? extends Number>

T 和？的区别：T 代表一个具体类型，当赋值的类型不确定的时候，我们用 通配符(?) 代替。？一般用于参数值是 未知类型 的容器类。
